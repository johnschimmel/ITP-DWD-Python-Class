Intro to Web Development

Web development is a general term for implementing a web site including web design, content creation, server configuration and programming. In this class we will explore web programming in its current state with open source tools and code and methods to serve and collect data from users.

Web programming is a two-sided operation including servers and clients.

Clients - making requests (mainly)

On the client-side, we are usually referring to a web browser. Typical usage, a user makes a request from their browser. The user's request goes out on the Internet to find a specific server and make a specific request to that server. A web server returns the appropriate content or information which is rendered into a structured visual web page by the browser.

Besides web browsers, a web client is any device or application that communicates with a web server. A Processing sketch or Arduino may be set up as clients to request the latest weather from Yahoo. The new wifi bathroom scales and thermostats are web clients as well, sending user data to a web server for later use. Best guess 99% of mobile phone application that require an Internet connection are connecting to a web server, they too are web clients.

Web clients usually expect to receive content from the web server in a specific format, browsers expect HTML, Instant messenger applications likely expect XML, many mobile phone applications expect JSON. It is up to the web client developer to decide on which format they require as a response from the web server.

Servers - accepting requests / return 'stuff'

A server is usually thought of as a powerful computer serving 1,000â€™s of user connections, but it is code that turns a powerful computer into a server. A server can be anything that is capable of running a piece of software to accept incoming connections from remote computers (or incoming requests from other applications).  Arduinos can become servers with an Ethernet of Wifi Shield. iPhones and Android can also run code to act as a server.

The web server is specifically designed to accept HTTP requests and return appropriate responses, HTML, XML, JSON and even file mime type .gif, .jpg, .mp4.

There are a variety of web servers, Apache popular in the PHP community. Servers are created in many programming languages too, Ruby, Python, Java and more recently JavaScript. A typical server will be a piece of code, or script that opens a socket. This socket acts like a doorway, allowing clients to request access to content and resources.

Python

Speaking personally, Python is one of my 2 favorite programming languages, the other being JavaScript. Python is nice to write, easy to read and kind to you if your codes breaks (well, kind enough).

Throughout the semester you'll get a sample of Python through code examples used in class and a chance to write your own with the assignments. Python's variables, lists (arrays), dictionaries (associative arrays) and classes are simple to learn and look very similar to other languages. The control syntax, if/else same as everywhere else. For-in loops and iterators are so nice you'll wish your other programming languages had them.

We'll be using a Python web framework called Flask (starting on week 3) for our web applications. It's very similar to Ruby's Sinatra and NodeJS's ExpressJS. Flask will let us create web servers in Python that can receive requests and return dynamic responses of HTML and JSON and static responses for images, css and javascript.

MongoDB

For our database, to save and retrieve data for our web applications we will use MongoDB, a document based database. It's popular for its schema-less, large scaling needs.  For us, it is the perfect tool to quickly save user submitted ( or sensor submitted) data. 

Heroku

For our web server we will use Heroku - a popular, free development cloud based web application hosting environment.  We will use GIT to push our code from our local computers to Heroku where it will the web application will be turned on and available to anyone on the Internet...in a matter of minutes.

Setting up Python & Development Tools

 

Install Python 2.7.5

Download and install latest Python (currently 2.7.5) DO NOT DOWNLOAD Python 3.x http://python.org/getit/releases/2.7.5/

  • Download the version for your operating system.
  • Open and click on the Python install package, it will install like a normal application.2. (FOR OS X) Update any existing Python command to the use latest (just installed) version.
  • On OS X, open your /Applications folder. Open the Python 2.7 folder. Double click on Update Shell Profile.command. This will allow your to access Python via the Terminal command 'python'.

Download & Install Python SetupTools

https://pypi.python.org/pypi/setuptools/1.1

Setuptools installs 'easy_install' command line tool - it's easy like the name says.

For OSX, here's the easy way

Run this command in your Terminal application, it will download ez_setup.py and install it with Python

curl https://bitbucket.org/pypa/setuptools/raw/bootstrap/ez_setup.py | python

When SetupTools have finished installing, download and install PIP and VirtualENV (it's easy now)

Install PIP and VirtualENV

  1. Install PIP

    In Terminal run the following command

    easy_install pip

    Check to see if installed, run the following command in Terminal. You should see a version number.

    pip -V

  2. Install VirtualENV

    easy_install virtualenv

    Check to see if installed, run the following command in Terminal

    virtualenv

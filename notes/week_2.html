Data structure - organzing your data

Lists

# list
missions = ['Gemini 3','Gemini 10','Apollo 10','Apollo 16','STS-1']

# add a mission
missions.append('STS-9') # inserting STS-9 into missions list

# access list item directly
print missions[1] # prints the second mission 'Gemini 10'

# get the length
print len(missions), " travelled missions"  # print the number of missions

Dictionaries

Just like associative arrays in PHP or hashlist in Java.

Dictionary is a container for Keys and Values.

# define dictionary and populate
myDictionary = {}
myDictionary['school'] = "ITP"
myDictionary['year'] = 2013
myDictionary['classes'] = ['DWD','Materials','Bicycles for the Mind','Puppets']

# dictionary define/populate one statement
mission_dates = {
  'Gemini 3' : '23 March 1965',
  'Gemini 10' : '18-21 July 1966',
  'Apollo 10' : '18-26 May 1969',
  'Apollo 16' : '16-27 April 1972 ',
  'STS-1' : '12-14 April 1981',
  'STS-9' : '28 November - 6 December 1983'
}

# retrieve mission
print mission_dates['Apollo 16']

# add to dictionary
mission_dates['MARS1'] = '25 December 2023'

# remove item from dictionary
del mission_dates['MARS1']

# retrieve the keys
just_the_mission_names = mission_dates.keys()  # a list of mission name

# retrieve the values
just_the_mission_dates = mission_dates.values() # a list of mission dates

 

Tuple - weirdo of the variables

Tuple is a data container, comma delimited list of variables between parentheses. Data can be retrieved in order.

favorite_color = ('Green','Brown','Orange')
first, second, third = favorite_colors 
print second # 'Brown'

 

# tuple
# tuples are weird - you can organize the basic variable types in a comma delimited list
# and retrieve them in that order
military_rank = ("Captain","US Navy")
rank, usforce = military_rank # rank = "Captain" and usforce = "US Navy"

Objects

Organizing many items with similiar data structure and functions.

# objects (classes)
# use objects when you have a lot of things with similiar data structure and actions
class Astronaut(object):

  # __init__ automatically called when creating an object
  def __init__(self, nameStr = None):
    print "Creating new astronaut:", nameStr
    self.name = nameStr

  def set_name(self, nameStr):
    self.name = nameStr

mr_awesome = Astronaut("J. Young")
mr_awesome.set_name("John Young")
mr_awesome.age = 81 #dynamically set object property/variable

print "This variable is :", type(mr_awesome)
print "Name: %s" % mr_awesome.name  # this is string formatting %s is a placeholder for a string

#print "%s is %d years old" % (mr_awesome.name, mr_awesome.age)
print mr_awesome.name, "is", mr_awesome.age, "years old"

Conditions and Loops

If Elif & Else

The basic IF statement

if 1 == 1:
  print "Yes, 1 == 1"

python_is_great = True
if python_is_great is True:
  print "Python is great!"

IF ELSE

name = "Tony Pony"

if name == "Tony Pony":
  print "Hello Tony Pony"
else:
  print "STRANGER DANGER!!"

IF ELIF and ELSE

name = "Tony Pony"

if name == "Tanya Perry":
  print "Hey Tanya"

elif name  == "Tony Pony":
  print "Still here Tony Pony?"

else:
  print "STRANGER DANGER!!"

Loops
